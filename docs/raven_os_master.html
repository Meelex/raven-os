OVERVIEW

Raven OS is a self-hosted home security and infrastructure network built entirely on Raspberry Pi hardware and a Windows workstation. Every piece of data stays on your hardware. No third-party cloud services. No API keys. No signups. The network monitors files, detects threats, locks malicious content, and requires physical human confirmation to unlock — all without touching the internet for decisions.

The network also hosts a full Egyptian-themed dungeon crawler (Gates of Darkness), a local AI (BitNet 3B), a biometric ring integration, and a WireGuard VPN with Hetzner relay — all running on the same sovereign infrastructure.

FILE LOCK / UNLOCK FLOW

1. File lands in Downloads (Legiom)
↓
2. Watchdog detects via folder watcher → SHA256 hash
↓
3. GET https://192.168.1.5:6174/lookup/:sha256
↓ SQLite hit → verdict · Cache miss → CIRCL hashlookup · Unknown → submit to MalwareBazaar anonymously
4. Verdict returned
├── CLEAN → file allowed, no action
└── MALICIOUS ↓
5. POST lock request → Duat :6176
↓
6. Duat SSHes as YourUser@Legiom → icacls /deny YourUser:RX
↓
7. Duat forwards alert → POST Raven :6175
↓
8. Raven e-ink: THREAT DETECTED · tap to dismiss → UNLOCK QUEUE
↓ Navigate: Menu → Unlock Queue → Select file → UNLOCK CONFIRM
9. Tap UNLOCK (left) or DENY (right)
↓
10. Raven POSTs decision → Duat :6176
↓
11. UNLOCK: Duat SSHes → icacls /grant YourUser:F → file accessible
DENY: File remains locked on disk
Watchdog also listens on port 6177 for direct unlock instructions. Local icacls lock/unlock via icacls /deny and icacls /remove:d + /grant.

PHILOSOPHY & KEY CONCEPTS

SOVEREIGN INFRASTRUCTURE

All data stays local. No third-party cloud. No API keys. No signups. Community feeds (MalwareBazaar) used anonymously. Unknown hashes submitted back anonymously — contributing without identifying. You own the hardware, you own the data.

SECURITY BY DESIGN

Authentication, encryption, and input sanitization built in by default — not added on request. Physical confirmation required for file unlock. No remote software bypass. The unlock chain requires a human hand on the device.

CONSENT-BASED DESIGN

"They have to hold hands." — The Anubis consent mechanism. Both rings must be physically present simultaneously to establish a monitoring connection. Cannot be initiated remotely. Vitals checked during handshake — if stress detected, connection blocked. Either party can revoke unilaterally. This is the ethical foundation of any future product.

SELF-HEALING NETWORK

Scarab detects Duat offline → falls back gracefully → reconnects when Duat returns. Fiancée never notices. Zero friction. The network heals around failures silently.

THE RAVEN RING

The COLMI R02 (right hand). Biometrics feed a rolling baseline — confidence score is a personal readiness indicator, not a system gate. Gesture vocabulary isolated: Iron House editing gestures only fire in editing mode. Hook left for fiancée — not built yet.
